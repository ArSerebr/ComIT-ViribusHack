# Task Assignment MVP

MVP для платформы студентов: модель анализирует задачи проекта и подбирает наиболее подходящего участника команды на основе эмбеддингов Sber GigaChat, совпадения скиллов и текущей загрузки.

## Что внутри

- `generate_data.py` генерирует два датасета:
  - `data/projects.json` - 100 проектов и задачи для их реализации
  - `data/team.json` - 18 участников команды с ролями и навыками
- `src/task_assigner.py` - модель назначения задач
- `main.py` - CLI для запуска по `project_id`

## Как запустить

```bash
python generate_data.py
$env:GIGACHAT_AUTH_KEY="your_authorization_key"
python main.py --project-id P001
python main.py --project-id P001 --format json
```

## Как работает модель

1. Для каждой задачи и каждого участника строится текстовый профиль.
2. Клиент автоматически получает access token у Sber по `GIGACHAT_AUTH_KEY`.
3. Профили переводятся в плотные эмбеддинги через `POST /api/v1/embeddings`.
4. Для каждой пары `задача -> участник` считается итоговый score:
   - semantic similarity по эмбеддингам
   - прямое пересечение требуемых и имеющихся навыков
   - бонус за подходящую роль
   - бонус за доступную емкость по времени
5. Для каждой задачи выбирается лучший кандидат, а в ответ добавляется объяснение "почему именно он".

## Переменные окружения

- `GIGACHAT_AUTH_KEY` - authorization key для получения токена
- `GIGACHAT_ACCESS_TOKEN` - опционально, если токен уже получен заранее
- `GIGACHAT_SCOPE` - по умолчанию `GIGACHAT_API_PERS`
- `GIGACHAT_EMBEDDING_MODEL` - по умолчанию `EmbeddingsGigaR`
- `GIGACHAT_VERIFY_SSL` - по умолчанию `false`

## Что можно улучшить дальше
